Most AI writing advice talks about tools and models. This is about the other half: the prompts.

The same model produces wildly different output depending on how you frame your request. A vague prompt gets vague output. A specific, well-structured prompt gets something you can actually use.

Below are templates for the writing tasks I use most often — plus some explanation of what makes each one work. Copy, adapt, and improve them for your context.

## The Core Principle: Specificity Is Everything

Before the templates, one rule to internalize: the more specific your prompt, the more useful the output.

"Write a blog post about coffee" → generic content you'll delete.
"Write a 1,200-word blog post for specialty coffee enthusiasts about why water temperature matters more than most people realize. Open with a surprising fact, use a conversational tone, and include 3 specific temperature recommendations with the reasoning behind each." → something worth editing.

The model knows a lot. Your job is to tell it exactly what you need.

## Email Templates

### Template 1: Drafting a Difficult Reply

```
I need to reply to this email: [paste email]

My position is: [what you want to communicate]
Tone: [professional and direct / warm but firm / apologetic but clear]
Key points to include: [list them]
What I want to avoid: [e.g., sounding defensive, over-promising, being vague on timeline]

Draft a reply of about [150-250] words.
```

**Why this works:** Difficult emails fail when you're not clear on your own position before writing. This prompt forces you to articulate it, which makes the output substantially better.

### Template 2: Cold Outreach Email

```
Write a cold email for the following situation:
- I'm reaching out to: [describe recipient — their role, company type]
- My ask is: [specific request]
- My credibility/reason they should care: [brief]
- Keep it under 150 words
- Subject line should be: [specific and non-spammy — e.g., not "Quick question" or "Following up"]
- Do not use flattery or say things like "I love your work" unless I specify genuine specifics
```

**Why this works:** Cold outreach fails most often because it's too long, too vague, or leads with the sender rather than the recipient's potential interest. This prompt builds those constraints in.

### Template 3: Summarizing a Thread for a Decision-Maker

```
I need to summarize this email thread for [role — e.g., my CEO, a client who wasn't involved]:

[paste thread]

Write a 4-6 sentence summary that covers:
1. What the issue was
2. What was discussed / decided
3. What the current status is
4. What action, if any, is needed from [the recipient]

Use plain language. No jargon.
```

## Blog Post Templates

### Template 4: Full Blog Post Draft

```
Write a blog post with the following parameters:

Topic: [specific topic]
Audience: [who they are, what they know, what they care about]
Goal of the post: [inform / persuade / teach a skill / entertain]
Tone: [conversational / formal / opinionated / neutral]
Length: approximately [word count]
Structure:
- Open with [a surprising fact / a counterintuitive claim / a concrete scenario]
- Include at least [N] specific examples
- End with [actionable next steps / a clear conclusion / an open question]

Do not start with "In today's world" or similar generic openers. Do not use filler phrases like "It's important to note that." Write like a knowledgeable human, not a text generator.
```

**Why this works:** The explicit instruction to avoid AI filler phrases makes a real difference. Also, specifying the opening type forces the model away from the generic "here's what I'll cover" lead.

### Template 5: Writing an Introduction for a Post You've Already Outlined

```
I've written an outline for a blog post. I need a strong opening section (about 150-200 words) that:
- Opens with a specific scene, question, or fact (not a generic statement)
- Creates genuine curiosity without being clickbait
- Establishes the tone as [casual and direct / authoritative / conversational]
- Leads naturally into the rest of the post

Here's my outline: [paste outline]

The reader is [describe audience]. Write the intro — don't explain it, just write it.
```

### Template 6: Headline Options

```
Write 10 headline options for a blog post about [topic].

Mix:
- Some that lead with the benefit
- Some that lead with the counterintuitive angle
- Some formatted as "How to X Without Y"
- Some formatted as questions
- At least 2 that are specific numbers (e.g., "7 ways to...")

Target reader: [brief description]
Tone: [casual / professional / punchy]

After the headlines, mark your top 3 with a *.
```

## Marketing Copy Templates

### Template 7: Product Description

```
Write a product description for [product name].

Key details:
- What it is: [one sentence]
- Main benefit: [the specific problem it solves]
- Who it's for: [target customer]
- Key features to mention: [list]
- Tone: [practical / aspirational / conversational]
- Length: [50-100 words / 150-200 words]

Avoid vague claims like "revolutionary" or "best-in-class." Be specific about what makes it useful.
```

### Template 8: Social Media Post (Multiple Variations)

```
Write 5 social media posts about [topic/announcement] for [platform — LinkedIn / Instagram / Twitter].

For each:
- Keep within platform limits ([LinkedIn: 1,300 chars / Twitter: 280 chars / Instagram: 2,200 chars])
- Vary the angle: make each post feel different (not the same message worded five ways)
- First post: lead with a question
- Second post: lead with a counterintuitive statement
- Third post: lead with a specific result or number
- Fourth and fifth: your choice of angle
- Include a CTA at the end of each
```

## Editing and Polishing Templates

### Template 9: Tighten Your Draft

```
Edit this for concision. Remove filler words, tighten sentences, and cut any phrase that doesn't add meaning. Keep the voice and all core ideas intact. Target about [X% shorter / 200 fewer words].

Here's the draft:
[paste text]
```

### Template 10: Rewrite for a Different Audience

```
I've written this for [original audience]. Rewrite it for [new audience], adjusting:
- Vocabulary and assumed knowledge
- Examples (swap to ones relevant to the new audience)
- Tone (if needed)

Keep the structure and core argument. Don't add new information.

Original: [paste text]
```

### Template 11: Write a Better Conclusion

```
This article needs a stronger conclusion. The current one is: [paste current conclusion or describe what you have]

The article covers: [2-3 sentence summary]
The key takeaway I want readers to leave with: [one clear sentence]

Write a conclusion of about [100-150] words. Close with something that gives the reader a concrete next step or leaves them thinking — not "In conclusion..." or a bland summary of what was already said.
```

## How to Get Better Results Over Time

Keep a personal prompt library. When a prompt produces great output, save it. When you figure out why a prompt failed, note that too. Your customized templates — trained on your specific use cases and writing voice — will outperform any generic list.

Give examples. For any template above, adding "Here's an example of the kind of output I want:" and pasting a piece of writing you like makes the result significantly better.

Iterate in the same conversation. "This is good, but make the opening punchier" or "Remove the third section — it's off-topic" — conversational refinement usually beats trying to write a perfect prompt on the first pass.
